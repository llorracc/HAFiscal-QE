## Then the QE template (HAFiscal-make/resources/qe/HAFiscal-QE.tex) is trivial:

```latex
% QE Submission Version
\documentclass[qe,nameyear,draft]{econsocart}
\RequirePackage[colorlinks,citecolor=blue,urlcolor=blue]{hyperref}

% Set up paths to find resources
\input{@resources/tex-add-search-paths}
\input{./.econtexRoot}

% Load HAFiscal packages and settings
\usepackage{@local/local}

\startlocaldefs
% Any QE-specific package settings
\endlocaldefs

\begin{document}

\begin{frontmatter}
% Include all QE metadata
\input{@local/qe-metadata}
\end{frontmatter}

% Main content (will be generated by consolidation script)
\input{HAFiscal-QE-content}

% Bibliography with QE style
\bibliographystyle{qe}
\bibliography{HAFiscal}

\end{document}
```

## Advantages of this approach:

1. **Zero Python parsing** - Just copy files and compile
2. **All metadata in one place** - Easy to find and update
3. **Uses existing infrastructure** - Works with @local/ path system
4. **Clean separation** - QE metadata separate from main document
5. **Simple QE build process**:
   - Copy HAFiscal-Public to working directory
   - Copy QE template as HAFiscal-QE.tex
   - Run consolidation to create HAFiscal-QE-content.tex
   - Run latexmk on HAFiscal-QE.tex
   - Done!

No extract-metadata.py needed at all!
